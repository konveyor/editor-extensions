# ISO/IEC 42001 — Control A.5.2 Evidence Ledger

**Control:** A.5.2 — Governance and lifecycle of AI systems
**Subject:** Prompt template governance, versioning, and automated validation for
Developer Lightspeed for MTA.
**Status:** Implemented — prompt templates decoupled, version-controlled, peer-review
mandated, and CI validation pipeline in place.

This ledger lists the direct file paths that constitute audit evidence. They are linked
into the central compliance ledger.

## 1. Structured versioning & decoupling

| Evidence                                                     | Path                                                                                |
| ------------------------------------------------------------ | ----------------------------------------------------------------------------------- |
| Dedicated prompt template directory (individual file assets) | [`prompts/templates/`](../../prompts/templates/)                                    |
| Semantic version + per-template manifest with checksums      | [`prompts/manifest.yaml`](../../prompts/manifest.yaml)                              |
| Render library (templates decoupled from app logic)          | [`prompts/src/`](../../prompts/src/)                                                |
| Governed package                                             | [`prompts/package.json`](../../prompts/package.json) (`@editor-extensions/prompts`) |

Prompt prose and control flow (loops/conditionals/per-language branching) live in the
template assets, rendered via `renderPrompt(id, context)`. Consumers no longer embed prompt
strings in compiled code — see `agentic/src/nodes/*` and `vscode/core/src/commands.ts`,
`vscode/core/src/modelProvider/modelProvider.ts`.

## 4. Scope boundary (for completeness)

The Kai **solution server** generates migration _hints_ using prompts that reside in the
separate [`konveyor/kai`](https://github.com/konveyor/kai) repository; they are consumed in
this repo only as opaque data via `agentic/src/clients/solutionServerClient.ts`
(`getBestHint`). That prompt surface is governed in `konveyor/kai`, not here. The
test-only evaluation prompt under `tests/kai-evaluator/` is part of the QE harness, not a
shipped/user-facing instruction set.
